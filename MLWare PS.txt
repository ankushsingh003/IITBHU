DATASET STRUCTURE
The dataset consists of short video clips extracted from action videos. Each
clip contains at most 150 frames and represents a continuous motion
sequence.
To create the challenge, the original videos have been intentionally
corrupted by modifying their temporal structure. Participants are required
to recover the correct chronological order of frames.
The dataset is divided into two subsets:
dataset/
|->train/
 |->video_0.mp4
 |->video_1.mp4
 |->...
|->test/
 |->video_5000.mp4
 |->video_5001.mp4
 |->...
|->train_labels.json
LABEL FORMAT
The labels are provided in JSON format.
Example:{
 "video_0": [5, 6, 7, 8, 9, 4, 3, 2, 1, 0]
 }
This list specifies the order in which frames from the corrupted video must
be selected to reconstruct the correct chronological sequence.
Each number represents the position of a frame in the corrupted video.
The reconstruction works as follows:
1st frame →  take the frame at position 5
 2nd frame →  take the frame at position 6
 3rd frame →  take the frame at position 7
 4th frame →  take the frame at position 8
...................
Thus, the JSON label specifies how the frames of the corrupted video must
be reordered to recover the true temporal sequence.
SUBMISSION FORMAT
Participants must submit their predictions in a CSV file with the following
structure:
video_id,order
 video_5000,"5 6 7 8 9 4 3 2 1 0"
 video_5001,"3 4 5 0 1 2"
Each row corresponds to one test video.
• video_id – the identifier of the test video
 • order – a space-separated list representing the predicted frame ordering
The numbers represent the positions of frames in the corrupted video.
The sequence describes the order in which frames should be selected to
reconstruct the correct chronological sequence.
EVALUATION METRIC: KENDALL RANK CORRELATION (KENDALL’S TAU)
Example
Ground truth:
 { "video_0": [2, 3, 4, 1, 0] }
Prediction:
 { "video_0": [2, 4, 3, 1, 0] }
All frame pairs (10 total):
 (2,3) (2,4) (2,1) (2,0) (3,4) (3,1) (3,0) (4,1) (4,0) (1,0)
Comparison:
 Correct pairs = 9  , Incorrect pairs = 1
Kendall Tau formula:
τ  = (C − D) / (C + D)
where, C = number of concordant pairs,  D = number of discordant pairs
Calculation:
τ  = (9 − 1) / (9 + 1)
 τ  = 8 / 10
 τ  = 0.8
Final leaderboard score = average Kendall Tau across all test videos.
DATASET SIZE
The dataset contains approximately: 5000 training videos & 100 test videos
Each video contains between 5 and 150 frames depending on the original clip
length.
Final Submission:

1.Model Weights
2.Working Codebase
3.Inference Script
4.A one-page document explaining the rationale behind the approach
5.requirements.txt or environment.yml for reproducibility
Naming Format:
TeamName_ModelRound
Submission Process:
Submissions will be accepted through the official Google Form.
A separate Kaggle competition will also be hosted where participants
must submit predictions for the test set.
The Kaggle leaderboard will be used to track performance and
rankings.
Important Rules
API calls are not allowed.
External datasets are allowed.
Pretrained open-source models are allowed.
Although there is no strict limit on model size, participants are
encouraged to design efficient and lightweight models with fast
inference pipelines.
We encourage participants to develop physics-aware and
interpretable deep learning approaches that capture the underlying
dynamics of motion in the videos. Solutions that rely purely on
simple heuristics or frame similarity may not generalize well. Instead,
methods that incorporate principles of temporal reasoning, physical
consistency, and interpretable modeling are strongly encouraged.
Teams that make a submission in Round 1 will be invited to present
their solutions before the judging panel.
Qualified teams must prepare and submit a presentation deck (PDF or
PPT) describing their approach.
The presentation should cover the following components:
1. Problem Understanding
2. Model Architecture and Methodology
3. Feature Engineering and Data Processing
4. Evaluation Strategy
5. Results and Insights
Presentation Format:
Each team will be given:
10 minutes for presentation
5 minutes for questions from the judges
Presentations will be conducted in offline mode 
Marking Scheme :
Evaluation will be based on the following criteria:
1. Innovation - 20%
2. Accuracy (Kendall Score) - 30%
3. Model Efficiency / Size - 20%
4. Code & Report Clarity - 10%
5. Presentation - 20 %
